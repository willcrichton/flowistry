1//! This crate provides the Flowistry API, a modular information flow analysis for Rust programs.
2//! The theory and evaluation of Flowistry is described in the paper ["Modular Information Flow through Ownership"](https://arxiv.org/abs/2111.13662) (Crichton et al. 2022).
3//! See [example.rs](https://github.com/willcrichton/flowistry/tree/master/crates/flowistry/examples/example.rs)
4//! for an example of how to use the Flowistry API.
5//!
6//! [Information flow](https://en.wikipedia.org/wiki/Information_flow_(information_theory))
7//! is whether one instruction or variable can affect another during a
8//! program's execution. Information flow can be used to analyze whether secure values
9//! can leak to insecure places (["information flow control"](https://www.cse.chalmers.se/~andrei/mod11.pdf)),
10//! and to analyze which parts of a program are relevant to a given variable (["program slicing"](https://en.wikipedia.org/wiki/Program_slicing)).
11//!
12//! This analysis uses the Rust compiler via the
13//! [rustc API](https://doc.rust-lang.org/nightly/nightly-rustc/).
14//! Given a [MIR](https://rustc-dev-guide.rust-lang.org/mir/index.html) [`Body`](https://doc.rust-lang.org/nightly/nightly-rustc/rustc_middle/mir/struct.Body.html),
15//! the function [`infoflow::compute_flow`] computes the information flow
16//! within that body.
17//!
18//! If you are interested in using Flowistry, please reach out directly to
19//! Will Crichton ([wcrichto@cs.stanford.edu](mailto:wcrichto@cs.stanford.edu)) or [join
20//! our Discord](https://discord.gg/XkcpkQn2Ah). If you use Flowistry in your research,
21//! then please cite our paper:
22//!
23//! ```bib
24//! @inproceedings{crichton2022,
25//!   author = {Crichton, Will and Patrignani, Marco and Agrawala, Maneesh and Hanrahan, Pat},
26//!   title = {Modular Information Flow through Ownership}, year = {2022},
27//!   isbn = {9781450392655}, publisher = {Association for Computing Machinery},
28//!   address = {New York, NY, USA}, url = {https://doi.org/10.1145/3519939.3523445},
29//!   booktitle = {Proceedings of the 43rd ACM SIGPLAN International Conference on Programming Language Design and Implementation},

31//!   location = {San Diego, CA, USA}, series = {PLDI 2022}, doi = {10.1145/3519939.3523445},
32//! }
33//! ```
